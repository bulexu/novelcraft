## 四、角色构建检查清单

```markdown
# 角色构建检查清单

角色名：
检查日期：

## 调色盘检查

- [ ] 有主色调、底色、对冲三个部件
- [ ] 每个部件都有具体衍生
- [ ] 衍生包含至少一条反直觉组合
- [ ] 有 3-5 句只有这个角色才会说的话
- [ ] 覆盖不同情绪状态

## 多面性检查

- [ ] 已判断是否需要多面性
- [ ] 如需要：每张面有五个部件
- [ ] 如不需要：已写明为什么不需要
- [ ] 面与面之间有过渡写法
- [ ] 有渗透提醒

## 二次解释检查

- [ ] 用自己的话解释了标签
- [ ] 写了"它不等于什么"
- [ ] 写了"AI 最容易误写成什么"

## 行为禁区检查

- [ ] 有 5-12 条禁区
- [ ] 每条禁区都有例外条件
- [ ] 标注了例外出现频率
- [ ] 写了破禁区后的反应

## 整体评估

- [ ] 遮住名字后，能从行为分出这个角色
- [ ] 角色有至少一个"只有他/她才会做"的东西
- [ ] 角色有明确的内逻辑
- [ ] 禁区是为了被打破的，不是为了锁死角色
```
