## 五、重场景检查卡

适用于：重逢、决裂、告白、牺牲、摊牌、和解

```markdown
## 重场景类型

- [重逢/决裂/告白/牺牲/摊牌/和解]

## 铺垫检查

- [ ] 信息铺垫：读者知道这件事迟早会来
- [ ] 情绪铺垫：角色已经绕着它活了很久
- [ ] 阻力铺垫：读者知道它发生要付代价
- [ ] 记忆铺垫：前面已有反复出现的动作/话/意象/承诺

## 写之前问

1. 读者知道它为什么重要吗？
2. 角色知道自己可能失去什么吗？
3. 前面至少预约过两次吗？
4. 发生之后，会带来明确后果吗？

## 如果有两条以上答不上来

→ 先补铺垫，不要急着把大场面写满
```

## 六、桥梁章模板

```markdown
## 桥梁章任务

- 收前一章的余波：
- 为下一次抬升存债：
- 调整关系温度：
- 让世界状态继续往前走：

## 不应该做的事

- 不要只是"过渡一下"
- 不要没有残留
- 不要让所有轴都停滞

## 完成后检查

- 这章结束后，有没有留下新的期待？
- 有没有推进至少一条情感债？
```

## 七、章尾检查清单

```markdown
# 章尾检查

## 变化检查

- [ ] 人物关系有变化
- [ ] 信息状态有变化
- [ ] 情绪温度有变化
- [ ] 局势压力有变化

（至少一项）

## 语言检查

- [ ] "不是X，是Y"句式 ≤ 1 处
- [ ] 没有直接心理描写（"她心里一酸"）
- [ ] 没有解释性动机从句
- [ ] 没有过度工整的排比

## 角色检查

- [ ] 每个核心角色遮住名字后还分得出来
- [ ] 没有角色全程只跑一个模式
- [ ] 角色没有把心里话直接说出来

## 残留检查

- [ ] 有明确的章尾残留
- [ ] 残留类型和上一章不同
```
