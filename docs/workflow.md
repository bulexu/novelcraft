# 创作工作流程

完整的小说创作流程，从构思到完稿。

## 四阶段创作模型

### 阶段 1：构思和规划

#### 1.1 定义创作核心

建立小说的核心价值观和主题。

**关键要素**：
- 核心主题和价值观
- 创作决策指南针
- 需避免的内容（廉价情感操纵、不必要暴力等）

#### 1.2 制定大纲

**最佳实践**：
- 从简单大纲开始（三幕结构或章节概要）
- 确定关键转折点和高潮
- 保持灵活性 - 允许情节演变

### 阶段 2：世界构建和角色塑造

#### 2.1 建立角色档案

使用 `/role create` 创建角色档案。

**关键要素**：
- 外貌、性格、动机
- 背景故事和创伤
- 角色弧线和成长轨迹
- 语言风格和习惯用语

#### 2.2 构建世界设定

使用 `/bg create` 建立世界观。

**记录内容**：
- 地理和地点
- 历史和文化
- 魔法系统或科技规则
- 社会结构和政治

### 阶段 3：场景创作

#### 3.1 使用行为推演

使用 `/infer` 推演角色行为，而非替角色决定行为。

**有效请求**：
```
场景：艾米在废弃工厂与追踪者对峙
情感：紧张、恐惧但决心坚定
重点：展示艾米的机智和勇气
长度：800-1000 字
```

**避免**：
- 过于宽泛的请求（"写第一章"）
- 缺乏情感方向
- 没有明确的场景目标

#### 3.2 追踪场景连续性

使用 `/plot` 系统维护情节一致性。

**功能**：
- 追踪角色状态和情感弧线
- 记录重要事件和时间戳
- 识别潜在的情节漏洞

### 阶段 4：审稿和修订

#### 4.1 一致性检查

使用 `/check full` 审查稿件。

**检查领域**：
- 角色一致性（外貌、性格、声音）
- 情节连续性
- 时间线准确性
- 主题一致性

#### 4.2 人性化润色

使用 `/check ai` 检测 AI 痕迹，参考 humanization.md 进行修复。

## 角色自主性原则

核心设计原则：AI 是角色行为的观察者和推演者，而非控制者。

### 推演优先级

1. 首先问：角色会怎么做？（考虑性格、经历、状态、动机）
2. 而非：情节需要角色怎么做？

### 处理"叛逆"时刻

当角色行为偏离作者预设：

| 情况 | 处理方式 |
|------|----------|
| 生命力体现 | 指出合理性，建议接受 |
| 逻辑漏洞 | 指出矛盾，建议合理化方式 |

### 判断依据

判断标准：这个行为是否让角色"更像他自己"？

## 增量创作原则

1. 一次专注于一个场景
2. 先完成粗稿，再精炼
3. 允许灵感自然涌现
4. 专注质量而非数量

## 常见陷阱

| 陷阱 | 解决方案 |
|------|----------|
| 过度依赖 AI | 你是作者，AI 是助手 |
| 跳过规划 | 一些结构能防止后期大返工 |
| 忽视一致性 | 小错误会累积破坏可信度 |
| 一次写太多 | 专注质量而非数量 |
