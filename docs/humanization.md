# 人性化处理指南

基于维基百科"AI 写作特征"指南 + 腾讯朱雀 AI 检测分析。

## 核心规则速查

**重要：不要过度简化！人类写作本来就有冗余、不完美。**

| 维度 | AI 特征 | 人类特征 | 策略 |
|------|--------|----------|------|
| 困惑度 | 低（选最常见词） | 高（用词多样） | 用不那么常见的词，**不要过度简化** |
| 突发性 | 低（句长均匀） | 高（长短交错） | **保留复杂句**，不要全拆短句 |
| 句法 | 过度规整，被动多 | 主动为主，**有轻微语病** | **保留一些人类冗余** |
| 词汇密度 | 低，避免术语 | 平衡高低频，适当术语 | **增加具体名词** |
| 段落连贯 | 过于线性 | 有跳跃性思维 | **允许逻辑跳跃** |
| 个性化细节 | 缺乏，泛泛而谈 | 有具体案例、数据 | **不要删除细节** |

## 中文特有问题

### 定语堆叠症

**这是中文 AI 写作最明显的问题**。AI 倾向于左分支结构，把修饰语全部堆在名词左边。

| AI 写法 | 人类写法 |
|--------|----------|
| 他看着那个**锈迹斑斑且散发着恶臭的**铁门 | 他看着那扇铁门，上面**锈迹斑斑**，散发着一股**恶臭** |
| 他按下了那个**因为挂机太久发烫的**手机屏幕 | 他按亮手机，挂机太久，屏幕已经有些**烫手**了 |
| 那几天没洗的油头、这标签页拉满的电脑 | 他的头发油了，大概几天没洗。电脑标签页拉满了 |

**核心原则**：
- 严禁名词前堆砌超过 6 个字的修饰语
- 先把物体写出来，再用独立短句描写状态
- 把"修饰语 + 的 + 名词"转化为"名词 + 状态/动作"

### "的"字结构限制

| AI 写法 | 人类写法 |
|--------|----------|
| 那个放在桌子上积满灰尘的杯子 | 杯子放在桌上，积满了灰尘 |

**规则**：一个分句中不要出现两个以上的"的"

### 通感误用

| 感官 | ❌ AI 错误 | ✅ 正确写法 |
|------|----------|-----------|
| 视觉 | "暖洋洋的侧脸"（触觉词用于视觉） | "阳光落在侧脸上，睫毛投下阴影" |
| 触觉 | "看起来温暖"（视觉+触觉混淆） | "声音很轻，带着笑意" |
| 听觉 | "响亮的颜色"（通感滥用） | "颜色很亮，刺眼" |

**判断标准**：
- 描写整体氛围/感觉 → 可用通感
- 描写具体视觉对象 → 用纯视觉词汇

### 视角一致性

**检查清单**：
- 从橱窗外路过 → 能否看清"挽在耳后的头发"？（需加"走近了能看清"）
- 从背后看 → 能否看到"眼角的红"？（不能）
- 黑暗中 → 能否看到"睫毛很长"？（不能，需有光源）

## AI 词汇黑名单

**高频 AI 词汇**：此外、与...保持一致、至关重要、深入探讨、强调、持久的、增强、培养、获得、突出、相互作用、复杂/复杂性、关键、格局、关键性的、展示、织锦、证明、宝贵的、充满活力的

**填充短语**：
- "值得注意的是" → 删除
- "不难发现" → 删除
- "可以说" → 删除
- "为了实现这一目标" → "为了实现这一点"
- "由于下雨的事实" → "因为下雨"

## 句式重构技巧

### 主动语态转换

| AI 写法（被动） | 人类写法（主动） |
|---------------|----------------|
| 数据显示这一趋势仍在继续 | 我们观察到这一趋势仍在继续 |
| 这一发现具有重要意义 | 这一发现改变了我们对问题的理解 |

### 非连续结构

| AI 写法（线性） | 人类写法（跳跃） |
|---------------|----------------|
| 因为 A，所以 B。由于 B，导致 C。因此 C 影响 D | A 发生了。至于 B——嗯，有点复杂。先说 C 吧... |

### 动词主导描写

| AI 写法（静态） | 人类写法（动词） |
|---------------|----------------|
| 他坐在**柔软的真皮**沙发上 | 他坐进沙发里，真皮的触感**陷下去**很软 |
| **巨大的**充值按钮 | 充值按钮**占据**了半个屏幕，刺眼地**闪烁**着 |

## 小说写作特有问题

### 高风险模式（AIGC值 > 0.9）

#### 1. 结构对称性过高

**问题**：AI 倾向于使用对称结构（排比、呼应），人类写作更随意。

| 对称类型 | ❌ AI 写法 | ✅ 人类写法 |
|---------|-----------|------------|
| 结尾排比 | "至少...至少...虽然...但..." | 打散为短句，删除排比标记 |
| 场景列举 | "有时候...有时候...有时候..." | 改为具体点位（周三、周五） |
| 对话对称 | 问→答→解释→反问→解释 | 打断对称，让对话跑题 |
| 情感递进 | "我想...但...至少..." | 用短句+自我质疑打断 |

**修复原则**：
- 遇到三个及以上平行结构 → 打断其中一个
- 遇到完整起承转合 → 删掉"承"或"转"的一部分
- 遇到对话完美对称 → 插入意外元素（数字、跑题、沉默）

#### 2. 时间/逻辑过度线性

**问题**：AI 倾向于完整交代因果链，人类写作会有跳跃和留白。

| 线性类型 | ❌ AI 写法 | ✅ 人类写法 |
|---------|-----------|------------|
| 时间过渡 | "过了...又过了...然后..." | 用短句跳过（"灯亮了。走了。"） |
| 因果链 | "因为A，所以B。由于B，导致C..." | "A发生了。C也来了。B？不管了。" |
| 心理推演 | "想起→不知道→但我想→至少我希望" | 用数字/细节打断，自我质疑 |
| 情节铺垫 | 完整的起承转合 | 省略中间环节，直接给结果 |

**修复原则**：
- 时间过渡超过两句 → 删除"过了"类词汇，用动作代替
- 因果链超过两环 → 断掉一环
- 心理活动超过三句 → 插入具体数字或自我否定

#### 3. 描写过于完整

**问题**：AI 倾向于把每个细节都写清楚，人类会有意识地省略。

| 完整类型 | ❌ AI 写法 | ✅ 人类写法 |
|---------|-----------|------------|
| 动作分解 | "他伸出手，握住把手，往下压，门开了" | "他推门进去。" |
| 情感解释 | "她很难过，因为想起了以前的事" | "她低着头。眼眶红了。" |
| 对话铺垫 | 完整的"开口→犹豫→说→反应" | 直接给对话，省略铺垫 |
| 场景全貌 | 从全景到细节完整交代 | 只写一个细节，其余留白 |

**修复原则**：
- 连续动作超过三个 → 合并为一个
- 情感后面跟"因为" → 删除解释，用动作暗示
- 对话前有超过两句铺垫 → 删除铺垫

### 低风险模式（AIGC值 < 0.4）

#### 1. 具体数字/细节

**原理**：数字增加困惑度，打断 AI 的常见模式。

| 类型 | 示例 |
|------|------|
| 时间精确化 | "两个小时零十分钟" vs "很久" |
| 数量具体化 | "你提了三次" vs "你一直在说" |
| 细节具体化 | "手指悬在输入框上，打三个字，删掉" vs "犹豫了很久" |

#### 2. 跳跃性思维

**原理**：模拟人类思维的跳跃和不完整。

| 类型 | 示例 |
|------|------|
| 自我质疑 | "两个月算吗？可能不算吧。" |
| 不完全回答 | "不知道。可能吧。" |
| 话题跳跃 | "她说...对了，今天的咖啡有点苦。" |

#### 3. 节奏打断

**原理**：打破句式的规整节奏。

| 类型 | 示例 |
|------|------|
| 短句打断 | "灯亮了。窗帘拉上。走了。" |
| 长短交错 | 在短句群中突然插入一个长句 |
| 标点变化 | 用句号打断本该连贯的句子 |

### 泛化修复策略

| 检测到的模式 | 修复动作 |
|-------------|----------|
| 三个及以上平行结构 | 删除或打断其中一个 |
| 完整的"因为...所以...因此" | 删掉"因此"，或断掉中间一环 |
| 对话完美对称（问答数相等） | 插入意外元素打断 |
| 时间过渡超过两句 | 用动作/短句替代 |
| 心理活动超过三句 | 插入数字/细节/自我否定 |
| 动作分解超过三个 | 合并为一个动作 |
| 情感后面有"因为" | 删除解释，用动作暗示 |
| 场景描写从全景到细节 | 只保留一个细节 |
